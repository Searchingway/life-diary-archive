项目名称（暂定）
本地优先的人生档案软件（Diary / Footprint / Reading Notes）

一、项目定位
做一个“本地优先、长期保存、可导入导出”的个人记录软件。
核心不是花哨社交，而是让用户可以长期稳定地记录：
1. 日记
2. 足迹
3. 读书笔记
4. 轻计划

项目目标：
- 先做出自己能真正每天使用的版本
- 以后可以扩展成 GitHub 开源项目
- 后续可考虑发布到应用市场

二、开发原则
1. 本地优先
- 所有数据默认保存在本地
- 不依赖账号系统和服务器
- 不把用户数据锁死在软件里

2. 纯数据可迁移
- 数据应尽量采用通用格式保存，如 JSON / Markdown
- 图片、附件单独保存
- 后续支持导出、备份、交换包导入导出

3. 先小后大
- 第一版只做最小可用功能
- 先不做云同步、账号系统、复杂 AI、复杂地图
- 先保证“能写、能存、能看、能导出”

4. 手机负责记录，电脑负责整理
- 手机端未来主要用于写日记、记录足迹、拍照
- 电脑端主要用于读书笔记、归档整理、导出

三、技术要求（当前建议）
1. 第一版优先使用 Python + PySide6 开发桌面版
2. 先做桌面版 MVP
3. 手机端后续再做，不要求第一版完成
4. 数据结构从一开始就要为未来多端交换预留接口

四、核心模块
1. 日记模块（第一优先级）
功能要求：
- 新建日记
- 按日期自动生成日记
- 编辑正文
- 自动保存 / 手动保存
- 左侧显示日记列表
- 可按日期浏览历史日记
- 支持插入图片
- 支持搜索标题或正文关键词

建议数据：
- 日期
- 标题（可选）
- 正文
- 图片列表
- 标签（后续可扩展）

2. 足迹模块（第二阶段）
功能要求：
- 先创建“地点足迹”
- 可填写地点名称和粗略描述
- 地点本身可插入图片
- 在某个地点下面再添加多个日期关联
- 每个日期关联都可以写一段感悟
- 每个日期关联都可以单独上传图片
- 足迹与当天日记双向关联
  1）从足迹跳转到对应日期日记
  2）从当日日记查看当天去过的地方

第一版足迹字段建议：
地点档案：
- id
- place_name
- summary
- images
- created_at
- updated_at

日期关联记录：
- id
- date
- thought
- images
- related_entry_id（可选）
- created_at
- updated_at

3. 轻计划模块（后续）
功能要求：
- 不是完整待办软件，只做轻计划
- 可以在日记中附带：
  1）今天完成了什么
  2）今天没完成什么
  3）明天最重要的三件事
  4）当前长期推进事项

注意：
- 先不要做复杂提醒、通知、重复任务、番茄钟

4. 读书笔记模块（后续）
功能要求：
- 记录书名、作者、阅读状态
- 可以保存书封面的图片用以显示
- 为每本书写笔记
- 支持关键词 / 标签
- 可以关联若干篇日记
- 重点是“书与我的关系”，不是单纯抄摘录

建议字段：
- title
- author
- status（想读 / 在读 / 读完）
- start_date
- finish_date
- tags
- summary
- notes
- related_entries

五、导入导出要求
1. 必须支持本地备份
2. 后续必须支持导出交换包
3. 交换包建议为 zip
4. 包内建议结构：

LifeArchiveExport_xxxx.zip
- manifest.json
- entries/
- footprints/
- books/
- media/

5. 每条记录必须有稳定 id
6. 每条记录必须有 created_at 和 updated_at
7. 后续导入时通过 id + updated_at 做合并基础
8. 文件修改时间戳只能作为辅助信息，不能作为唯一合并依据
9. 可以导出为word和PDF

六、数据存储建议
建议每条记录独立存储，避免全塞进一个数据库。
示例：

Diary/
  entries/
    2026-04-24-entry001/
      entry.json
      content.md
      img1.jpg
  footprints/
    place001/
      footprint.json
      summary.md
      images/
        img1.jpg
      visits/
        visit001/
          visit.json
          thought.md
          images/
            img2.jpg
  books/
    book001/
      book.json

要求：
- 结构清晰
- 手工也能备份
- 软件没了，原始内容仍然可读

七、第一版（MVP）必须完成的内容
第一版只做以下内容：
1. 主窗口
2. 日记列表
3. 新建日记
4. 编辑日记
5. 本地保存
6. 本地读取
7. 按日期浏览
8. 基础搜索（可选）
9. 基础图片插入（可选，若时间不够可延后）

第一版明确不要做：
- AI 总结
- 多端实时同步
- 地图视图
- 云服务
- 登录注册
- 漂亮动画
- 复杂富文本编辑器
- 足迹高级统计

八、界面要求（第一版）
界面尽量简单实用，不追求花哨：
左侧：
- 日记列表
- 新建按钮

中间：
- 标题输入框（可选）
- 正文编辑区

上方或下方：
- 保存按钮
- 导出按钮（后续）
- 插图按钮（后续）

九、项目版本规划
v0.1
- 纯日记功能
- 能写、能存、能读

v0.2
- 增加足迹功能
- 日期与足迹双向关联

v0.3
- 增加轻计划功能
- 每篇日记附带简单计划区

v0.4
- 增加读书笔记功能

v0.5
- 增加交换包导入导出
- 支持手机记录、电脑整理的数据流

十、项目亮点（后续可写进 README / 简历）
1. 本地优先
2. 长期归档
3. 数据可迁移
4. 日记 + 足迹 + 阅读笔记一体化
5. 面向真实个人生活记录，而不是花哨社交
6. 支持未来 AI 总结和人生档案整理

十一、给 AI 编码时的要求
1. 一次只实现一个小闭环
2. 先复述需求，再写代码
3. 输出可运行代码，不要只给伪代码
4. 代码要有清晰文件结构
5. 不要擅自加入额外复杂功能
6. 优先保证能运行、能保存、能读取

十二、当前最重要的开发目标
先把第一版做活：
- 可以新建一篇日记
- 可以保存到本地
- 可以重新打开查看
只要这一步做通，项目就算真正开始了。

十三、当前实现进度（2026-04-24）
当前桌面版已经完成日记 + 足迹 + 读书笔记的当前版本：
- 新建日记
- 左侧列表查看历史日记
- 打开并编辑已有日记
- 保存更新到本地
- 删除日记
- 按日期 / 标题 / 正文搜索
- 插入图片并保存到当前日记目录
- 支持给图片填写备注名
- 导出当前日记为 Word 和 PDF
- 导出时显示进度条
- 支持按开始/结束日期导出多天日记
- 导出时按旧日期在前排序
- 第二篇及后续日记自动另起一页
- PDF 直接由 Word 文档转换生成，保证与 Word 使用同一版式
- 导出时不写入创建时间和更新时间，只保留日记日期
- 图片可填写备注名；只有填写了备注名，导出时才显示图片标题
- 已增加足迹页签
- 支持新建、查看、编辑、保存、删除足迹
- 足迹支持按日期 / 地点 / 感悟搜索
- 足迹支持插入图片、移除图片、打开图片、图片备注名
- 足迹数据保存到独立的 footprints 目录
- 支持从日记查看当天足迹
- 支持从足迹打开当日日记
- 当前日记与足迹联动以按日期查询为主
- 当前足迹结构调整为“地点档案 + 多个日期关联记录”
- 地点档案可填写地点主题、粗略描述和地点图片
- 每个日期关联可单独填写感悟、上传图片，并按日期打开对应日记
- 足迹元数据中保留可选 related_entry_id 作为后续增强关联字段
- 已增加读书笔记页签
- 支持新建、查看、编辑、保存、删除读书笔记
- 读书笔记支持书名、作者、阅读状态、开始日期、完成日期、标签
- 支持填写阅读摘要和读书笔记正文
- 支持添加书封或其它书籍图片
- 支持关联若干篇已有日记
- 支持从读书笔记页直接打开关联日记
- 读书笔记支持按书名 / 作者 / 标签 / 笔记 / 关联日记搜索

当前本地数据落地方式：
- 每篇日记一个独立目录
- 目录内包含 entry.json、content.md 和可选的 images 子目录
- 每条足迹一个独立目录
- 目录内包含 footprint.json、summary.md、地点 images 子目录
- 每个地点目录下面还包含 visits 子目录，里面按日期关联记录继续拆分 visit.json、thought.md 和 images
- 每本书一个独立目录
- 目录内包含 book.json、summary.md、notes.md 和可选的 images 子目录
- 每条日记保留稳定 id、created_at、updated_at
- 每条地点档案和每条日期关联记录同样保留稳定 id、created_at、updated_at
- 每条读书笔记同样保留稳定 id、created_at、updated_at
- 删除操作当前采用软删除：写入删除标记并从活动列表隐藏
